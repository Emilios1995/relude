Relude_ResultIor

This module does not have a toplevel documentation block.

type t('a, 'e) = | IOk('a) | IError('e) | IBoth('a, 'e)

ResultIor is similar to Belt.Result, but it has the ability to collect "non-fatal warning" information during applicative validation.

E.g. if you are doing applicative validation to construct a User model, you could parse a phone number and allow it, but collect a warning that certain phone number formats are deprecated.

let pure: 'a => t('a, 'e)

let ok: 'a => t('a, 'e)

let error: 'e => t('a, 'e)

let both: ('a, 'e) => t('a, 'e)

let map: ('a => 'b, t('a, 'e)) => t('b, 'e)

let apply: ( t('a => 'b, 'e), t('a, 'e), ('e, 'e) => 'e ) => t('b, 'e)

let bind: (t('a, 'e), 'a => t('b, 'e)) => t('b, 'e)

let flatMap: ('a => t('b, 'e), t('a, 'e)) => t('b, 'e)
